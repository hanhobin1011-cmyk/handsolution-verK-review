# 리뷰 피드백 원장

이 파일은 사용자 검수, 내부 검수, 전체 리뷰에서 나온 피드백 중 상태 변경이나 다음 재생성 지시로 이어지는 항목만 모아 두는 중앙 원장이다. 세부 수식 검산, 그래프 스펙, 문항별 긴 분석은 기존 패킷과 테스트 문서에 남기고, 이 원장에는 추적에 필요한 핵심만 기록한다.

상태의 원천은 `assets/output-registry.json`이고, 현재 대기열은 `REVIEW_QUEUE.md`로 생성한다. `OUTPUT_INDEX.md`는 registry에서 만든 읽기용 생성물이다. 이 파일은 자동 생성물이 아니므로 피드백이 생길 때 직접 갱신한다.

## 운영 규칙

- 사용자 피드백이 산출물 상태를 바꾸면 이 원장에 먼저 한 줄을 추가한다.
- 상태를 `기준 후보`, `최종 후보`, `최종본`으로 올릴 때는 사용자 승인 표현과 대상 파일을 함께 기록한다.
- 상태를 `수정 필요`, `이전 후보`, `폐기`로 내릴 때는 실패 원인과 다음 작업번호 반영 지시를 적는다.
- 즉시 재생성이 필요한 항목은 기존 PNG 부분 보정이 아니라 새 작업번호 전체 재생성으로 처리한다.
- 원장 갱신 후 `assets/output-registry.json`, 생성물 `OUTPUT_INDEX.md`, 관련 패킷/테스트 문서, `REVIEW_QUEUE.md`가 같은 상태를 가리키는지 확인한다.
